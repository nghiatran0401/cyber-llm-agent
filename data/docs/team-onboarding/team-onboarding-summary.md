# Team Onboarding Summary (Read This First)

## What This Project Is

This is a cybersecurity multi-agent assistant with:

- backend API (FastAPI)
- agent runtime (G1 single-agent + G2 multi-agent)
- web app (Next.js)
- sandbox/lab for simulated attacks

Main goal for the next 4 weeks: improve quality and reliability while each member owns one clear track.

## Current Reality (Honest Status)

- The system works and can be demoed.
- Core mechanisms are implemented.
- Some parts are still inconsistent and need hardening (tests, contracts, integration behavior).
- Team success depends on strict ownership and clean integration.

## Team Structure (5 People)

- Person 1: Platform Lead (infra, CI/CD, API contracts, release quality)
- Person 2: ReAct Owner (reasoning loop, trace behavior, stop policy)
- Person 3: RAG Owner (retrieval quality, citations, RAG reliability)
- Person 4: Memory Owner (session memory, recall relevance, persistence)
- Person 5: Tooling Owner (CTI, log parser, sandbox/lab reliability)

## Baseline -> 4-Week Target Scores

- Platform: 6.5 -> 8.5
- ReAct: 6.0 -> 8.0
- RAG: 4.5 -> 8.0
- Memory: 6.5 -> 8.2
- Tooling: 6.0 -> 8.0
