# Threat Modeling for Web Applications

## Workflow
1. Define assets and trust boundaries.
2. Map entry points and data flows.
3. Identify threats (STRIDE/abuse cases).
4. Define mitigations and verification tests.
5. Revisit after major design changes.

## High-Risk Areas
- Authentication and session flows
- Admin functions
- File processing pipelines
- Third-party integrations and webhooks
